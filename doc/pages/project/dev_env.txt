====== Development environment ======

Linux host is used as development and build environment for ADK-based applications.

''adk_env_setup'' script is used to set up ADK environment (see "tools" directory in ADK sources). ''adk_sudo'' script used to raise privileges in ADK environment. 

===== Build tools =====

The following packages are used:
  * **binutils** from http://ftp.gnu.org/gnu/binutils/
  * **gmp** from http://gmplib.org/
  * **mpfr** from http://www.mpfr.org/
  * **mpc** from http://www.multiprecision.org/
  * **flex** from http://flex.sourceforge.net/
  * **bison** from http://ftp.gnu.org/gnu/bison/
  * **gcc** from http://gcc.gnu.org/
  * **doxygen** from http://sourceforge.net/projects/doxygen/files/
  * **w32api** from https://sourceforge.net/projects/mingw/files/MinGW/Base/w32api/
  * **mingwrt** from https://sourceforge.net/projects/mingw/files/MinGW/Base/mingw-rt/
  * **avr-libc** from http://download.savannah.gnu.org/releases/avr-libc/

All environment components are built and installed automatically by ''adk_install.py'' script. Directory with the archived packages should be provided to the script.

Should be installed on build machine via package manager:
  * gcc
  * g++
  * texinfo
  * m4
  * gawk
  * gcc-multilib
  * linux-libc-dev
  * libc6-dev-i386
  * libc6-dbg
  * libfontconfig1-dev
  * libncurses5-dev
